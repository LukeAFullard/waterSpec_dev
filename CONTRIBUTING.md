# Contributing to waterSpec

First off, thank you for considering contributing to `waterSpec`. It's people like you that make open source such a great community.

## Where do I go for help?

If you are having issues, please let us know by opening an issue on the [project's issue tracker](https://github.com/example/waterSpec/issues).

## How do I contribute?

We welcome contributions in many forms. The most common ways to contribute are by reporting bugs and submitting code changes.

### Reporting Bugs

If you find a bug, please open an issue on our issue tracker. Please include the following information:

-   A clear and descriptive title.
-   A detailed description of the problem, including the steps to reproduce it.
-   The version of `waterSpec` you are using.
-   Any relevant error messages or tracebacks.

### Submitting Changes (Pull Requests)

If you would like to contribute code to fix a bug or add a new feature, please follow these steps:

1.  **Fork the repository** on GitHub.
2.  **Create a new branch** for your changes. Please use a descriptive branch name (e.g., `fix-plotting-bug`, `add-wavelet-feature`).
3.  **Make your changes.** Ensure that you adhere to the project's coding style (see `pyproject.toml` for `black` and `ruff` configurations).
4.  **Add or update tests** for your changes. We aim for high test coverage.
5.  **Submit a pull request** to the `main` branch of the `waterSpec` repository.
6.  **Write a clear pull request description** explaining the "what" and "why" of your changes.

Thank you for your contribution!
